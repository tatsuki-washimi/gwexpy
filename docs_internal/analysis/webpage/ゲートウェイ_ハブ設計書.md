### hero 画像の要件

- **ファイル名**: `docs/_static/images/hero_plot.png`
- **内容**: `FrequencySeriesMatrix` に対してフィッティングを適用した出力図

  - カラーマップを使い、複数チャンネルの周波数応答が視覚的に映えること
- **サイズ**: 幅 800px、高さ 400〜500px、PNG、白背景
- **生成元候補**: `case_bootstrap_gls_fitting.ipynb` または `advanced_fitting.ipynb`

## 3. 言語切替の設計

### 現状
- `_templates/layout.html` の `sidebartitle` ブロック内に `#language-switcher-box` として実装済み
- **対応するページへの直接遷移ロジックは既に実装されている**（`pagename.split('/ja/')` による pathto 計算）
- 問題: サイドバー最下部に埋もれており視認性が低い

### 改善後
- 配置: **ヘッダー右端に固定（`position: fixed`）**
- デザイン: 丸角ピル型バッジ（`border-radius: 16px`）
- 動作: 現在のページの対応言語ページへ直接遷移（既存ロジックを流用）
- サイドバー下部の `#language-switcher-box` は削除

## 4. 実装上の注意事項

### Sphinx RTD テーマの制約
- `sphinx_rtd_theme` ではサイドバーを CSS で非表示にするのが最も確実（`.wy-nav-side { display: none !important }`）
- ゲートウェイ専用スタイルは `raw:: html` ブロック内に直接埋め込む、または gateway専用 CSS クラスで分岐する
- `|release|` と `|today|` は Sphinx が自動解決する（`conf.py` から取得）

### 英語版ハブ（`web/en/index.rst`）
- 日本語版と**構造は完全に対称**に保つ
- カード名・説明文は英訳し、リンク先は `gwexpy_for_gwpy_users_en` に切り替える

### toctree 構成
現行の4グループ（導入 / 学習 / 高度な機能 / リファレンス）は**変更しない**。
ページ本文の構成変更が主なスコープ。
