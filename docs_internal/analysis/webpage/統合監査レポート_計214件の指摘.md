## 方針（確定）

### 実施優先度
1. **第1優先**: 矛盾の解消（Pythonバージョン、用語不統一、壊れたリンク）
2. **第2優先**: 導線・構造改善（サイドバー再編、チュートリアルメタデータ）
3. **第3優先**: コンテンツ拡充（API Example、比較画像、比較表）
4. **第4優先**: 技術基盤（ファビコン、OGP、レスポンシブ対応、Plotly等）

### 検証方法
- **ソースファイル（.rst / .ipynb）** と **ブラウザ表示の両方** で確認する。

**再検証注記（2026-04-16）**: `conda gwexpy` 環境で再ビルドした HTML（`/tmp/gwexpy-docs-verify-conda-escalated`）を基準に現状を再確認した。Section 1-2（ゲートウェイ/ハブ）は現在編集中のため、今回の状態更新対象から除外する。

**進捗反映注記（2026-04-17）**: Phase 1-5 相当の機械的改修（インストール/クイックスタート/学習ロードマップ/GWpy移行ガイド/I/Oガイド/時刻/数値安定性/ScalarField/検証済みアルゴリズム/アーキテクチャ/チュートリアル一覧/リファレンス入口）の状態更新を本レポートへ反映した。

**作業報告（2026-04-17）**: `user_guide/io_formats.html` の 4グループ再編を JA/EN 両方へ実装し、`NDS2` / `GWOSC` を A: 重力波標準形式へ統合した。あわせて `user_guide/interop.html` を JA/EN で新規作成し、I/O から分離した interop 専用の公開入口を追加した。いずれも `conda run -n gwexpy sphinx-build -b html docs` で再ビルド確認済み。

### スコープ外（申し送り事項）
- API実装レベルの変更が必要な指摘（例: N85 Zarrの1Hz自動割当）は本タスクの対象外とし、別途 Issue として管理する。

### 共通用語ルール（確定）
- **ライブラリ名の正式表記**: `GWexpy`（ドキュメント・本文）
- **Pythonモジュール名**: `gwexpy`（コード・インポート文）
- `GWExPy` は誤表記として全廃する。

## 8. ファイル I/O ガイド (`io_formats.html`)

> [!IMPORTANT]
> **グループ設計更新・実装待ち**: `io_formats` の設計は [I/O フォーマット グループ設計](./io_formats_グループ設計.md) に切り出し済み。ここでいう「対応」は end-user の `.read()` / `.write()` に限定する。設計書は現在、内部モジュール名ではなく公開 API を基準に、全管理対象を `掲載済み / 実装済み（未掲載） / 実装済み（一部対応） / 対応中 / 対応予定` で状態管理する。`to_*` / `from_*` や ROOT / xarray / Zarr / NetCDF4 / JSON / SQLite の変換系は [interop / 変換対応設計](./interop_変換対応設計.md) に分離した。実装前に両書を参照すること。

### グループ設計の確定内容（要約）

4グループに分類。`io_formats` 本文では direct I/O のみを扱い、interop は別管理とする。再編後の要約表は、一般ユーザーが選択判断に使う主表示対象だけを抜粋し、未掲載実装や placeholder は設計書側で別表管理する。`NDS2` と `GWOSC` は A: 重力波標準形式に含め、備考欄で `ネットワーク経由` と明示する。

| グループ | 主な形式 | 主な対応クラス |
|---|---|---|
| A: 重力波標準形式 | GWF, HDF5, ndscope-hdf5, DTTXML, NDS2, GWOSC | TimeSeries系、HDF5 は Field を含む |
| B: 地震・地球物理観測 | MiniSEED, SAC, GSE2, K-NET, WIN / WIN32, ATS, ATS.MTH5, MTH5 standalone | TimeSeries, TimeSeriesDict |
| C: 汎用・解析用 | CSV / TXT, NetCDF4, Zarr, Pickle, ROOT | TimeSeries系中心。Pickle は全クラス、ROOT は EventTable 直I/O |
| D: 計測機器・ロガー専用 | GBD, TDMS, SDB / SQLite / SQLite3, WAV, MP3, FLAC, OGG, M4A | TimeSeries, TimeSeriesDict |

> [!NOTE]
> **D1（未解決）**: `ndscope-hdf5` の形式名を `hdf5.ndscope` に統一するかどうかは API 破壊的変更を伴うため別途判断が必要。
> `ROOT`, `NetCDF4`, `Zarr` の変換系や Field 橋渡しはこのセクションの設計対象外とし、別紙 interop 設計で管理する。
> 実装済みだが未掲載の `ndscope-hdf5`, `SQLite`, `SQLite3`、および placeholder reader を持つ `ORF`, `MEM`, `WVF`, `WDF`, `TAFFMAT`, `LSF`, `LI` と周波数系列側 placeholder は、I/O 設計書で明示列挙して状態管理する。
