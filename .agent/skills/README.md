# gwexpy Agent Skills

**総スキル数**: 39

Agent Skills は AI エージェントに専門知識と実行能力を提供するオープンスタンダード形式のパッケージです。

## Quick Start

最も頻繁に使用するスキル:

| スキル          | 説明                                            | 使用場面           |
| --------------- | ----------------------------------------------- | ------------------ |
| `finalize_work` | 作業完了時の検証・整理・コミット                | 作業終了時         |
| `run_tests`     | pytest, GUI, ノートブックテスト                 | コード変更後       |
| `fix_errors`    | MyPy, Python 3.9 互換性, ノートブックエラー修正 | エラー発生時       |
| `manage_docs`   | ドキュメントのビルド・同期                      | ドキュメント更新時 |

## Categories

### ワークフロー管理

| スキル             | 説明                                                                                   |
| ------------------ | -------------------------------------------------------------------------------------- |
| `finalize_work`    | 作業完了時の検証・整理・コミットを一括実行する（quick/full モード対応）                |
| `handover_session` | AIモデル間や作業セッション間での円滑な引継ぎのためのドキュメント・プロンプトを作成する |
| `prep_release`     | バージョン更新、CHANGELOG整備、パッケージビルドなど、リリース前の準備を行う            |

### 開発・コーディング

| スキル                    | 説明                                                                    |
| ------------------------- | ----------------------------------------------------------------------- |
| `add_type`                | gwexpyに新しい配列型（Array/Series/Field）とコレクションを実装する      |
| `extend_gwpy`             | GWpy/Astropyクラスの安全な継承と拡張のためのガイドライン                |
| `fix_errors`              | MyPy、Python 3.9 互換性、ノートブックなど各種エラーを修正するパターン集 |
| `manage_field_metadata`   | 多次元フィールドの4D構造維持、ドメイン変換、物理単位の整合性を管理する  |
| `manage_gui_architecture` | PyQt等のモノリシックなGUIクラスを分離・リファクタリングする             |
| `refactor_nb`             | Jupyter Notebook内のコード要素を一括置換・リファクタリングする          |
| `refactor_skills`         | スキルの整理（統廃合・分離・分類の更新）を行う                          |

### 品質保証・テスト

| スキル       | 説明                                                                   |
| ------------ | ---------------------------------------------------------------------- |
| `run_tests`  | pytest、GUIテスト、ノートブック実行を含む統合テストスイートを実行する  |

## Progressive Disclosure

スキルは3層構造で設計されています:

```
Tier 1: メタデータ（起動時）
├─ name + description (~100トークン)
│
Tier 2: 指示（スキル有効化時）
├─ SKILL.md 本文 (<500行推奨)
│
Tier 3: リソース（必要時のみ）
└─ reference/*.md, scripts/, assets/
```
