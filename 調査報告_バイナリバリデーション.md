# バイナリファイル読み込み時のサンプル数バリデーション調査報告

## 概要
`.ats` などのバイナリファイルにおいて、ヘッダーに記録されたサンプル数と実際のファイル内のデータサイズが一致しない場合にエラー（`ValueError`）を発生させるソースコードとその対象フォーマットを特定しました。

## 特定されたファイルフォーマットとソースコード

### 1. Metronix ATS フォーマット (`.ats`)
*   **ソースコード**: [gwexpy/timeseries/io/ats.py](file:///home/washimi/work/gwexpy/gwexpy/timeseries/io/ats.py)
*   **該当箇所**: `_read_timeseries_ats_file` 関数 (L138-142)
*   **ロジック**:
    ```python
    data_raw = np.fromfile(f, dtype=data_dtype, count=total_samples)
    if data_raw.size != total_samples:
        raise ValueError(
            f"ATS data block shorter than expected: got {data_raw.size}, expected {total_samples}"
        )
    ```

### 2. GRAPHTEC GBD フォーマット (`.gbd`)
*   **ソースコード**: [gwexpy/timeseries/io/gbd.py](file:///home/washimi/work/gwexpy/gwexpy/timeseries/io/gbd.py)
*   **該当箇所**: `_read_data_block` 関数 (L374-378)
*   **ロジック**:
    ```python
    expected = header.counts * n_channels
    raw = fh.read(expected * header.dtype.itemsize)
    array = np.frombuffer(raw, dtype=header.dtype, count=expected)
    if array.size != expected:
        raise ValueError("GBD data block shorter than expected")
    ```

## その他
以下の形式についても確認しましたが、同様の厳密なサンプル数バリデーション（バイナリブロック長によるチェック）はプロジェクト内では実装されていませんでした：
*   **WIN/WIN32 (NIED)**: `win.py` 内で ObsPy ベースの読み込みを行っており、ワーニングの発生やパッチ適用による対応が行われています。
*   **WAV**: `wavfile.read` (SciPy) に依存しており、ライブラリ側で処理されます。
*   **TDMS**: `nptdms` ライブラリに依存しています。
*   **SDB (SQLite)**: データベース形式であり、SQLクエリで読み込まれます。
