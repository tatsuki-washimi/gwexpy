# `.harness/` ポリシー計画

**作成日:** 2026-04-25
**関連 issue:** #190
**ステータス:** 草案（人間レビュー待ち）

---

## 背景

`.harness/` は AI エージェント向けのガイドライン・スキル・フック・ルールなどを格納するプロジェクト標準ディレクトリである。
PR #189 で絶対パス露出を最小修正した上で、本プランではファイルの追跡方針を整理する。

---

## 判断基準

### 追跡継続（チーム共有資産）の条件

- 絶対パス・個人識別情報を含まない
- チーム全員または AI ツール全般が恩恵を受けるルール・定義
- `conda run -n gwexpy` のようにプロジェクト標準として合意済みの値のみ使用

### 非追跡対象（gitignore 候補）の条件

- 個人のローカル絶対パスを含む
- 個人の API キー・トークン・資格情報
- プロジェクト標準と異なる conda 環境名など個人固有の上書き設定
- 特定の AI ツールのみが参照する一時生成ファイル

## ファイル別分類

| パス | 分類 | 理由 |
|------|------|------|
| `AGENTS.md` | **追跡継続** | チーム共有の中核ドキュメント |
| `README.md` | **追跡継続** | ハーネス概要 |
| `agents/*.md` | **追跡継続** | エージェント定義、ツール中立 |
| `config/quality-gates/` | **追跡継続** | 品質ゲート設定、環境依存なし |
| `hooks/hooks.json` | **追跡継続** | 絶対パスなし、`conda run -n gwexpy` はプロジェクト標準 |
| `rules/common/*.md` | **追跡継続** | プロジェクト物理・テストルール |

**削除対象:** 現時点でなし。全ファイルが共有価値を持つ。

**テンプレート化対象:** 現時点でなし。`hooks.json` は既にポータブルな形式。

## 将来の個人設定分離：`.harness.local/` パターン

現在は全ファイルが共有可能だが、以下のシナリオで個人設定が必要になりうる：

- conda 環境名が `gwexpy` でない開発者が `hooks.json` を上書きしたい
- 特定ツールのみ有効にしたい hooks を追加したい
- 実験的なエージェント定義をローカルのみで保持したい

**方針案:**

```
.harness.local/          ← gitignore 対象（新設・今後の準備）
├── hooks/
│   └── hooks.json       ← hooks.json の個人上書き（オプション）
└── agents/              ← 個人実験用エージェント定義
```

- `.harness.local/` は `setup_symlinks.sh` が認識し、存在すれば `.harness/` より優先してシンボリックリンクを設定する
- `.harness.local/` は `.gitignore` 対象であり、個人設定ファイルはコミットしない

---

## `.gitignore` 更新案

```gitignore
# .harness のローカル上書き設定（個人環境依存）
.harness.local/
```

これ以外の `.harness/` ファイルは追跡継続のため、追加の ignore 行は不要。

---

## 再混入防止策

絶対パスや個人設定の再混入を防ぐため：

1. `hooks.json` 編集時は `git diff .harness/hooks/hooks.json` で絶対パスが含まれていないことを確認する
2. `setup_symlinks.sh` は相対パスのみ使用し、絶対パスを引数に取らない設計を維持する
3. `.harness/rules/common/harness-editing.md` のチェックリストで、PR 前に `.harness/` への個人設定混入を確認する
