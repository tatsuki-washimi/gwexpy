# Webドキュメント改善提案

`docs/` および `gwexpy/` コードベースの徹底的な監査の結果、ドキュメントの構造は健全であり、AI生成の不自然な痕跡もなく、最近のAPI統一（`fftlength`/`overlap`）とも完全に整合していることが確認されました。

ドキュメントの品質とユーザー体験をさらに高めるため、以下の包括的な改善提案を実施することを検討してください。

## 1. `gwexpy_for_gwpy_users_ja.md` の英訳

現在、日本のユーザー向けに書かれた優れた移行ガイド（`docs/web/ja/user_guide/gwexpy_for_gwpy_users_ja.md`）が存在します。これはGWpyに対するGWexpyの拡張機能（SeriesMatrix、Field API、高度なHHT/STLTメソッド、I/Oフォーマットなど）を明確に説明しています。
**提案**: 国際的なGWpyユーザー層に向けた英語版（`gwexpy_for_gwpy_users_en.md`）を作成します。現在、英語版の `getting_started.md` はクイックスタートへのリンクで補っていますが、専用のガイドの作成を強く推奨します。

## 2. インタラクティブなノートブックの導入 (Colab / Binder)

`/tutorials` フォルダには充実したチュートリアルが含まれています。
**提案**: Jupyter Notebookチュートリアルの上部に "Open in Colab" や "Launch Binder" のバッジを追加します。これにより、新規ユーザーの環境構築の負担が軽減され、ブラウザから直接 `Field API` や `SpectrogramMatrix` といった機能を試すことができるようになります。

## 3. APIリファレンスとチュートリアルの相互リンク

Jupyterチュートリアルには豊富なコンテキストが含まれていますが、`docs/web/*/reference/` にあるAPIリファレンスは機能的な説明にとどまっています。
**提案**: 複雑なメソッド（`bode()`、`hht()`、`bootstrap_spectrogram()`など）のソースコード内のAPI Docstringを強化し、その機能を示す特定のチュートリアルへの Sphinx `:doc:` リンクまたは Markdown リンクを追加します。これにより、「関数が何をするか」と「実際にどのように使うか」のギャップを埋めることができます。

## 4. アーキテクチャと物理学の深掘りページ

新しい `Field` API（4次元物理学サポート）と MIMO 解析向けの `SeriesMatrix` 構造により、ライブラリは高度な物理シミュレーションの領域に入りつつあります。
**提案**: チュートリアルを超えて、基盤となる数学とドメイン管理（時間/周波数 ↔ 空間/波数）を説明する「アーキテクチャと物理」セクションを作成し、現在 `ALGORITHM_CONTEXT.md` に記載されている知見をパブリックな Web ドキュメント向けに最適化して公開します。

## 5. チュートリアルのユニットテストカバレッジ拡大

Jupyter Notebookは主要なユーザー向け例であるため、基礎となるAPIが変更された場合（最近の `/fs` 統一などで見られたように）動作しなくなるリスクがあります。
**提案**: プルリクエスト（PR）のたびに機械的にすべてのチュートリアルノートブックをテストするビルドパイプライン（`pytest-notebook` や `nbval` などのツール）を CI/CD に統合し、公開されているドキュメントが常に正しく動作するコードを反映していることを保証します。
