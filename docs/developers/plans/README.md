# Developer Plans Index

This directory separates current coordination documents from historical plans
and audit manifests.

## Layout

- `active/`: current coordination documents and plans that still guide upcoming
  work.
- `archive/contract-audits/`: completed or superseded issue-specific audit
  plans and contract baselines.
- `archive/release-and-docs/`: completed release and documentation planning
  notes.
- `manifests/`: audit manifests and PR evidence records.
- `notes/`: coordination notes that are useful context but are not active
  implementation plans.

When adding a new plan, keep only active coordinating documents at the top of
`active/`. Move completed issue-specific plans to `archive/` after the related
PR or issue is closed.
