# アーキテクチャとデータフロー

`gwexpy` の設計思想と、内部的なデータ処理の流れについて解説します。
本パッケージは GWpy を拡張し、多チャネルの時系列行列操作や 4次元物理フィールドの扱いを直感的に行えるよう設計されています。

## データ構造の設計思想

### 1. 行列オブジェクトの平坦化フロー

`TimeSeriesMatrix` や `FrequencySeriesMatrix` は、内部で scikit-learn 等の機械学習ライブラリと親和性の高い形式への変換を自動で行います。
通常、(チャネル数, 時間/周波数数, 行列の列数) といった 3次元的なデータを、メタデータを保持したまま一時的に 2次元へ平坦化し、計算後に元の次元と GPS タイムスタンプを復元します。これにより、高度なアルゴリズムを物理情報を失わずに適用可能です。

### 2. 4次元フィールド (Field) モデル

`ScalarField` は (Time, Frequency, x, y) の 4次元構造を基本単位とします。
インデクシング操作において**次元を削減しない**（4次元を維持する）ことで、グリッド情報とデータの整合性を常に担保しています。

## 主要な数値解析エンジン

### 独立成分分析・主成分分析 (ICA/PCA)

スカイキットラーン (scikit-learn) のエンジンを利用していますが、`gwexpy` は以下の部分を強化しています：

- **振幅スケーリング**: ICA 実行前後のデータ振幅を内部で単位分散に標準化し、物理量を維持したまま収束性を向上。
+
- **メタデータ再接続**: 分離された独立成分に対して、元の時系列が持っていた GPS 時刻やサンプリングレートを再割り当てします。

### 高速コヒーレンスエンジン (:term:`BruCo`)

`FastCoherenceEngine` は、数千のチャネルを対象としたノイズ監査を高速化するために設計されています。
共通のターゲット信号に対する FFT/PSD をキャッシュし、補助チャネルごとに最小限の計算のみを行うことで、標準的な `coherence()` 呼び出しに比べて数倍〜数十倍のパレート改善を実現しています。

### ベイズ・フィッティング (GLS/MCMC)

非対角成分に相関を持つエラー（共分散行列が既知の場合）を扱う一般化最小二乗法 (GLS) をコアに持っています。
`emcee` との統合により、複素数値データ（伝達関数等）に対しても、実部・虚部の相関を正しく考慮した多次元パラメータ推定が可能です。
