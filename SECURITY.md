# Security Policy

`gwexpy` のセキュリティを確保するためのポリシーです。

## サポートバージョン

現在、以下のバージョンの `gwexpy` に対してセキュリティアップデートを提供しています。

| バージョン | サポート状況 |
| :--- | :--- |
| >= 0.1.0 | :white_check_mark: サポート対象 |
| < 0.1.0 | :x: 非サポート |

## 脆弱性の報告方法

`gwexpy` にセキュリティ上の脆弱性を見つけた場合は、公の Issue を作成せず、以下の手順で報告してください。

1. GitHub リポジトリの **Security** タブに移動します。
2. **Advisories** を選択します。
3. **Report a vulnerability** ボタンをクリックして、詳細を送信してください。 (GitHub の Private Vulnerability Reporting)

報告を受けた後、5 営業日を目安に初期対応の回答を行います。

> [!IMPORTANT]
> スパム防止のため、開発者のメールアドレスは非公開としています。
> セキュリティ上の重大な問題については、上記の GitHub Private Vulnerability Reporting を通じて報告してください。
>
> GitHub のアカウントをお持ちでない方や、より詳細な情報の提供を希望される方は、[GitHub Discussions](https://github.com/gwexpy/gwexpy/discussions) からご連絡ください。

## セキュリティ上の注意事項

### `pickle` の利用について

`gwexpy` は GWpy との互換性や一部のメタデータ保存のために `pickle` を使用する場合があります。
`pickle` は信頼できないソースからのデータを読み込む際に、任意のコードが実行されるリスクがあります。

- **信頼できないソース（インターネット経由で入手した正体不明のファイルなど）から提供された `pickle` データを `gwexpy` で読み込まないでください。**
- `gwexpy` では、より安全な代替形式として **HDF5** または **JSON** の利用を推奨しています。

## サプライチェーンセキュリティ

`gwexpy` では、依存ライブラリの脆弱性を迅速に検知し対応するため、以下の対策を講じています。

### 自動更新の導入

- **Dependabot**: GitHub の Dependabot を使用し、`pip` (Python) および GitHub Actions の依存関係を毎週自動でスキャンしています。
- **手動レビューポリシー**: 自動でマージを行うのではなく、すべての更新 PR をメンテナーが手動でレビューし、テスト（CI）をパスすることを確認した上でマージします。

### セキュリティスキャン

- 各プルリクエストおよび週次で、`pip-audit`, `bandit`, `CodeQL` による自動セキュリティスキャンを実行しています。スキャンの結果は GitHub の **Security** -> **Code scanning** から確認可能です。

---

## 謝辞

セキュリティ上の問題を発見し、責任を持って報告してくださった方々には、修正後のリリースノート等で謝辞を掲載させていただく場合があります。
